    フィッシュボーン図（石川図）¶

    複雑な問題が発生したとき、その根本原因は単一ではなく、複数の異なる分野にまたがる相互に関係した要因の結果であることが多いです。直感だけに頼ると、いくつかの重要な潜在的原因を見落とす可能性があります。フィッシュボーン図（石川図）は、考案者である石川馨博士にちなんで名付けられた、強力な視覚的根本原因分析ツールです。この図の主な目的は、チームが特定の問題（結果）につながるすべての潜在的原因を体系的かつ包括的にブレインストーミングし、整理するための構造化されたフレームワークを提供することです。

    フィッシュボーン図の魅力はその構造化されたブレインストーミングプロセスにあります。この図は、チームが事前に定義されたさまざまな視点から考えるための古典的な原因カテゴリ（魚の「主な骨」）を提供し、思考の盲点を回避します。「人、機械、材料、方法、環境」などのすべての可能性のある要因を1つの図に示すことで、チームは問題の複雑さについて包括的かつ全体的な理解を得ることができます。その上で、さらに深く調査・検証する価値がある主要な原因を特定することができます。これは、チームの散らばったアイデアを論理的で秩序ある「問題の全体像」に整理するための優れたツールです。

    フィッシュボーン図の構造¶

    フィッシュボーン図は、視覚的かつ固定的ないくつかの部分から構成されています：

    • 魚の頭：図の右端に位置し、通常は四角で囲まれており、分析したい問題または結果が記載されています。例えば、「今月の製品不良率が20%増加」など。
    • 背骨：魚の頭から左に向かって伸びる水平のメインライン。
    • 主な骨：背骨から斜めに伸びるいくつかの主要な分岐で、問題に寄与する主要な原因カテゴリを表します。これらは思考の枠組みを提供します。
    • 小枝／細い骨：主な骨から伸びる小さな分岐で、チームメンバーが各主要カテゴリの下でブレインストーミングした、より具体的な潜在的原因を表します。

    古典的な原因分類モデル（主な骨）¶

    分析対象の業界に応じて、フィッシュボーン図の主な骨の分類として異なる古典的なモデルを柔軟に選択できます：

    • 製造業で一般的に使われる「6M」モデル:

      • Manpower（人）: 作業者のスキル、経験、責任感、疲労など。
      • Machine（機械）: 生産設備の老朽化、精度、メンテナンス状況など。
      • Material（材料）: 原材料の品質、仕様、サプライヤーの安定性など。
      • Method（方法）: 作業手順、操作マニュアル、工程パラメータの設定など。
      • Measurement（測定）: 測定器の精度、検査基準、データ記録の正確さなど。
      • Milieu/Mother Nature（環境）: 職場の温度、湿度、照明、組織文化など。

    • サービス業界で一般的に使われる「4S」または「8P」モデル:

      • 4S: サプライヤー、システム、周囲、スキル。
      • 8P: 製品（商品／サービス）、価格、流通チャネル、プロモーション、人材、プロセス、物理的証拠、生産性と品質。

    フィッシュボーン図の描き方と使い方¶

    1. ステップ1: 「魚の頭」（問題）を明確に定義する チームとともに、分析する問題について明確で具体的かつ曖昧でない合意を形成します。この問題文をホワイトボードの右側の「魚の頭」の位置に記載します。

    2. ステップ2: 「背骨」と「主な骨」（原因カテゴリ）を描く メインラインを描き、業界や問題の特性に応じて適切な分類モデル（例：6M）を選び、いくつかの主な骨を描き、カテゴリ名を記載します。

    3. ステップ3: 「小枝／細い骨」（具体的な原因）をブレインストーミングして埋める

      • これがフィッシュボーン図の核心部分です。ファシリテーターがチームを導き、各主な骨のカテゴリごとに、考えられるすべての具体的な原因をブレインストーミングします。
      • キーテクニック: 各具体的な原因の下で、「5Why分析」を組み合わせて連続的に「なぜ？」と質問することで、より深い原因を掘り下げます。例えば、「人」の主な骨において、「従業員の操作ミス」という原因があるとします。「なぜミスが起きたのか？」と問うことで、「トレーニング不足が原因」というより深い原因にたどり着き、それを細い骨として描くことができます。
      • 考え出されたすべての原因を、対応する主な骨に小枝または細い骨としてつなげます。

    4. ステップ4: フィッシュボーン図を分析し、主要な原因を特定する フィッシュボーン図が完成すると、問題の原因の全体像が示されます。この時点でチームは図全体を一緒に見直し、議論や投票、または単純なデータ検証を通じて、問題の発生に最も可能性が高く、あるいは最も大きな影響を持つ「重要少数原因」を特定します。これらの主要な原因は、別の色のペンで囲むことができます。

    5. ステップ5: 次の検証と改善のための計画を立てる フィッシュボーン図自体はブレインストーミングと分析のツールであり、直接問題を解決することはできません。次に、チームは囲まれた主要な原因に対して具体的な検証計画を立てます（例：現場に赴き、仮説が正しいかデータを収集して検証する）、その上で最終的な改善策を策定します。

    実際の適用例¶

    事例1: 「アプリのクラッシュ率が高い」問題の分析

    • 魚の頭: 最新バージョンをリリースしたアプリで、ユーザーのクラッシュ率が50%上昇。
    • 主な骨: ソフトウェア開発モデルのバリエーションを使用。例：コード、ビルド、テスト、環境、人。
    • 分析: チームのブレインストーミングにより、「コード」の主な骨の下で「新しいサードパーティライブラリにメモリリークがある」ことが判明。また、「テスト」の主な骨の下では「自動テストケースが低価格Android端末をカバーしていない」ことが判明。さらなるデータ検証の結果、チームは「メモリリーク」がクラッシュの主原因であることを特定しました。

    事例2: 「マーケティングキャンペーンの成果が悪い」問題の分析

    • 魚の頭: 今年の「ダブルイレブン（独身の日）」プロモーションの売上が目標に達成しなかった。
    • 主な骨: マーケティングの4Pモデルを使用：製品、価格、流通、プロモーション。
    • 分析: チームは「価格」の主な骨の下で「クーポンのルールが複雑すぎてユーザーが理解できない」ことを発見。「流通」の主な骨の下では「ソーシャルメディア広告がターゲット層に正確に届いていない」ことが判明。この分析を通じて、チームは次回キャンペーンに向けて明確な「落とし穴回避ガイドライン」を提供できます。

    事例3: 「術後感染率の増加」問題の分析

    • 魚の頭: 今四半期の整形外科病棟の術後感染率が前四半期比で5%増加。
    • 主な骨: 医療分野向けの6Mモデルを使用。
    • 分析: 医師、看護師、感染管理専門家からなる多職種チームが共同でフィッシュボーン図を作成しました。「方法」の主な骨の下で「術前の皮膚消毒手順の実施不足」、「環境」の主な骨の下で「病棟の換気システムフィルターの交換が遅れている」ことが最も疑わしい主要な原因として特定されました。次に、これらの2点に焦点を当ててデータ収集と現場観察を行います。

    フィッシュボーン図の長所と課題¶

    主な長所

    • 構造的かつ包括的: チームが複数の次元から体系的に考えるための明確な構造を提供し、見落としを効果的に回避します。
    • チーム参加と合意形成を促進: 優れたチーム協働ツールであり、問題の複雑さについて全員の知恵を集め、合意を形成できます。
    • 視覚的: 複雑な因果関係を非常に直感的かつ明確に提示します。

    潜在的な課題

    • 過度に複雑になる可能性: 非常に複雑な問題では、フィッシュボーン図が非常に大きくごちゃごちゃになり、明確な焦点を失うことがあります。
    • 原因の重みを反映できない: フィッシュボーン図自体は、どの原因がより大きな影響を持っているかを示しません。パレート分析などの他のツールと組み合わせて優先順位を決定する必要があります。
    • 「仮説」の集まりにすぎない: フィッシュボーン図上のすべての「原因」は、データで検証されるまではあくまで「潜在的・疑わしい」原因であり、事実ではありません。

    拡張と関連ツール¶

    • 5Why分析: フィッシュボーン図の黄金のパートナーです。フィッシュボーン図を使って横方向・広範囲な原因のブレインストーミングを行う一方で、5Why分析は特定の原因に対して縦方向・深掘りの根本原因分析を行うことができます。
    • ブレインストーミング: フィッシュボーン図はブレインストーミングのための構造化されたフレームワークを提供し、アイデアの生成をより秩序立てて焦点を当てたものにします。
    • パレート分析: フィッシュボーン図ですべての潜在的原因を特定した後、データを収集し、パレート分析を使って問題の80%を引き起こしている「重要少数原因」を特定できます。

    参考文献: 石川馨博士は日本の戦後品質管理運動のパイオニアの一人です。彼が考案した「7つの品質管理ツール」の一つであるフィッシュボーン図は、世界中の品質管理および継続的改善活動で広く使用されており、トータル品質管理（TQM）およびシックスシグマの実践において不可欠な基本ツールです。
